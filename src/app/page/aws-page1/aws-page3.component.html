Being able to configure and update your environments is an important part of DevOps. Doing these tasks quickly and easily is even more important. Ansible helps solve the problem of manually configuring servers one-by-one. Using Ansible as an automation tool, system administrators can update and configure servers from a single machine. This course is a way to learn how to set up Ansible and a sandbox environment to learn in.

You’ll learn how to manage servers using Ansible playbooks, stand up new environments, and monitor your cloud infrastructure. Basic understanding of server configuration and unix commands is required.

In this course, we cover:

  • The history of Ansible, and why it is used today.
  • Some of the other orchestration tools available in the market like Chef, Puppet and Saltstack.
  • How to set up Ansible locally and configure local environment to use Ansible.
  • Learn how to read Ansible commands and build playbooks.
  • Learn how to stand up a simple infrastructure from nothing to fully functional configured environment.
  • Learn how to get started with Ansible, one of the most popular configuration management tools out there. If you’re passionate about DevOps, Ansible is a great tool to get started with.
